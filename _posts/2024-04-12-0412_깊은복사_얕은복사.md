### 깊은 복사와 얕은 복사

우선 dart에서 모든 데이터는 reference type 이다.

### flutter 의 copyWith 메서드

- 불변 객체의 일부 필드만을 변경해서 새로운 객체를 만들때 사용됨.

```
User copyWith({ String name, int age}) {
    return User(
        name : name ?? this.name,
        age: age ?? this.age,
    );

}

```

### 깊은복사 (deep copy)

객체를 복사할때 해당 객체가 포함하는 모든 하위 객체까지 새롭게 복사하는것.
아예 새로운 객체를 만들어 내는것.
원본 객체와 복사된 객체 사이에는 어떠한 참조 관계도 존재하지 않게 됨.
(바꿔말하면 원본을 바꿔도 복사된 객체에 영향을 끼치지 않음)
=> 실제값을 새로운 메모리 공간에 복사하는것

### 얕은 복사 (shallow copy)

객체 복사시 최상위 필드만 복사.
복사된 객체 내의 하위 객체를 변경하면 원본 객체에도 영향을 미침
=> 주소값을 복사함

### 그럼 copyWith 메서드가 깊은 복사인가?

구현 방식에 따라 달라짐.
copyWith 의 기본 목적은 객체의 일부 속성을 변경하여 새로운 객체를 생성하는 것임.
깊은복사인지 얕은복사인지는 구현에 따라 다름..
위 예시에 든 User 클래스 안에 구현된 copyWith는 깊은 복사 특성을 가짐.
왜냐면 String, int 같은 기본 데이터 타입은 불변이므로.
즉 새로운 객체에 할당된 값은 원본 객체의 값을 변경해도 영향을 받지 않음.
하지만 완전 깊은 복사인지 확인하려면 User 객체가 가지고 있는 다른 속성들에 달려있음
User가 객체나 리스트같은 컬렉션을 속으으로 가지고 있고, 현재 구현된 copyWith 로 복사된다면 이는 얕은 복사로 간주됨.

### 직렬화

데이터를 저장하기 위해서 포맷을 변경하는 과정.
직렬화 과정에서 객체를 항상 새로운 객체를 생성한다.

### dart에서는 깊은 복사를 기본적으로 제공해주지 않음.

필요할때를 판단해서 직접 구현해야 한다.
-> 복사 생성자나 팩토리 메소드를 통해서 가능.

하지만 다트 언어는 불변성(immutability)를 지향하고 있지 않나?
-> 다트에서 불변성은 갸벌저거 의도적으로 설계를 통해 구현해야 하는 개념임.
예를들면 final 을 사용해서 변수 재할당을 막거나, 불변 컬렉션 사용하거나 등등
하지만 깊은 복사는 자동으로 이루어지지 않으므로 내가 명시적으로 써야함.
